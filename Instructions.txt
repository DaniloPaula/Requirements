pip freeze > requirements.txt
pip install -r D:\Projetos\Requirements\requirements.txt --use-deprecated=legacy-resolver

--use-deprecated = legacy-resolver makes it possible to use the old conflict checking mechanism between packages,
which can save a lot of time

For each new package installed, I add it to the end of the reqs.txt file.
For example "pandas>=1.0.0" without quotes. With that he will always get the most current version of each package

"python -m pip list --outdated --format columns" List outdated packages with column formatting

For init Git repository:
1) git init
2) git remote add origin url/to/your/fork
3) git remote -v
4) git add .
5) git commit -m "message"
6) git push -f origin master

For update/remove packages in Ubuntu WSL:
sudo -- sh -c 'apt-get update; apt-get upgrade -y; apt-get dist-upgrade -y; apt-get autoremove -y; apt-get autoclean -y'

For update anaconda:
1) conda update conda
2) conda update anaconda
3) conda update --all

For update from pip:
pip --disable-pip-version-check list --outdated
